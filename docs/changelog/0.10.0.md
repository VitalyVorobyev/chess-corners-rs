# 0.10.0

A breaking release that lands a single consistent configuration tree
for the detector across every binding layer (Rust, Python, WASM, CLI
JSON).

## Detector configuration

- `DetectorConfig` carries a `strategy: DetectionStrategy` enum that
  picks ChESS or Radon and holds the detector-specific tuning inline.
  Top-level fields (`threshold`, `multiscale`, `upscale`,
  `orientation_method`, `merge_radius`) apply uniformly to both
  detectors.
- `ChessConfig` is the ChESS strategy payload. It carries `ring`
  (`Canonical | Broad`), `descriptor_ring`, NMS / cluster thresholds,
  and a `refiner: ChessRefiner`.
- `RadonConfig` is the Radon strategy payload. It carries
  `ray_radius`, `image_upsample`, `response_blur_radius`, `peak_fit`,
  NMS / cluster thresholds, and a `refiner: RadonRefiner`.
- Refiners are per-detector: `ChessRefiner` lists `CenterOfMass`,
  `Forstner`, `SaddlePoint`, and optionally `Ml`. `RadonRefiner`
  lists `RadonPeak` and `CenterOfMass`. The type system rules out
  cross-detector mismatches.
- Every enum-with-payload knob uses the same shape: `Threshold`,
  `MultiscaleConfig`, `UpscaleConfig`, `ChessRefiner`, and
  `RadonRefiner` all serialize as externally tagged JSON with a
  payload object (or no payload for unit variants).

## Threshold / multiscale / upscale

- `Threshold::Absolute(f32)` or `Threshold::Relative(f32)` replaces
  the previous threshold encoding. `Absolute(0.0)` encodes the ChESS
  paper's `R > 0` contract; `Relative(f)` accepts a fraction of the
  per-frame response maximum.
- `MultiscaleConfig::SingleScale` skips the pyramid;
  `MultiscaleConfig::Pyramid { levels, min_size, refinement_radius }`
  enables coarse-to-fine detection. Honoured by both detectors.
- `UpscaleConfig::Disabled` skips the pre-pipeline upscaler;
  `UpscaleConfig::Fixed(factor)` upscales by 2, 3, or 4 before
  detection. Output corner coordinates are always returned in the
  input pixel frame.

## CLI

- `chess-corners run <config.json>` consumes the new `DetectorConfig`
  shape inside a CLI runner envelope (`image`, `output_json`,
  `output_png`, `log_level`, `ml`).
- Per-flag overrides: `--threshold-absolute` / `--threshold-relative`,
  `--chess-ring`, `--descriptor-ring`, `--chess-refiner`,
  `--radon-refiner`, `--upscale-factor`, plus the multiscale tuning
  flags.

## Python

- `chess_corners.DetectorConfig` exposes the typed tree directly.
  Canonical factory methods: `chess()`, `chess_multiscale()`,
  `radon()`, `radon_multiscale()`. The deprecated aliases
  `single_scale()` and `multiscale_preset()` forward to the new names
  and will be removed in 0.12.0.
- Tagged classes mirror the Rust enums: `Threshold.absolute(...)` /
  `.relative(...)`, `MultiscaleConfig.single_scale()` /
  `.pyramid(...)`, `UpscaleConfig.disabled()` / `.fixed(k)`,
  `ChessRefiner.center_of_mass(...)` / `.forstner(...)` /
  `.saddle_point(...)` / `.ml()`, `RadonRefiner.radon_peak(...)` /
  `.center_of_mass(...)`.
- Each tagged class exposes a `kind` getter (variant tag) and a
  `payload` / `value` getter (active variant's tuning).
- All public config objects support `to_dict()` / `from_dict()` /
  `to_json()` / `from_json()` / `pretty()` / `print()`. Unknown keys
  in `from_dict` are rejected with a clear `ConfigError`.
- The detector entry point is `chess_corners.Detector(cfg).detect(img)`.
  The free-function `find_chess_corners` is removed.

## WebAssembly

- The JS package surfaces the typed configuration tree:
  `DetectorConfig`, `ChessConfig`, `RadonConfig`, `ChessRefiner`,
  `RadonRefiner`, `MultiscaleConfig`, `UpscaleConfig`, `Threshold`,
  and their factory functions
  (`ChessRefiner.withForstner(cfg)`,
  `ChessRefiner.withCenterOfMass(cfg)`,
  `ChessRefiner.withSaddlePoint(cfg)`,
  `RadonRefiner.withRadonPeak(cfg)`,
  `RadonRefiner.withCenterOfMass(cfg)`,
  `MultiscaleConfig.singleScale()`,
  `MultiscaleConfig.pyramid(levels, minSize, refinementRadius)`,
  `UpscaleConfig.disabled()`,
  `UpscaleConfig.fixed(factor)`,
  `DetectionStrategy.fromChess(...)`,
  `DetectionStrategy.fromRadon(...)`).
- `ChessDetector` reads and writes its full configuration through the
  typed tree: `detector.getConfig()` returns a `DetectorConfig`
  snapshot, `detector.applyConfig(cfg)` commits edits. Nested edits
  via getters propagate to the live detector.

## Ergonomic builder surface

- `DetectorConfig::chess()` and `DetectorConfig::chess_multiscale()`
  are the canonical preset constructors for ChESS detection.
  `single_scale()` and `multiscale()` remain as deprecated aliases that
  forward to the new names and will be removed in 0.12.0.
- Closure-based strategy mutators `with_chess(|c| ...)` and
  `with_radon(|r| ...)` allow targeted in-place edits to the strategy
  payload without destructuring the outer `DetectorConfig`.
- Top-level chainable setters on `DetectorConfig`:
  `with_threshold`, `with_multiscale`, `with_upscale`,
  `with_orientation_method`, `with_merge_radius`.
- Shortcut constructors on strategy-specific types:
  - `ChessRefiner::center_of_mass()`, `ChessRefiner::forstner()`,
    `ChessRefiner::saddle_point()`.
  - `RadonRefiner::radon_peak()`, `RadonRefiner::center_of_mass()`.
  - `MultiscaleConfig::pyramid_default()` (levels 3, min\_size 128,
    refinement\_radius 3); `MultiscaleConfig::pyramid(levels, min_size,
    refinement_radius)` for explicit tuning.
- Python bindings expose equivalent surface: closure-style mutation
  maps to keyword-argument overrides (`cfg.with_chess(refiner=ChessRefiner.forstner())`).
- WASM bindings expose equivalent surface via options-object form
  (`cfg.withChess({ refiner: ChessRefiner.withForstner({}) })`).

### Migration guide (builder layer)

| Before | After |
| ------ | ----- |
| `DetectorConfig::single_scale()` | `DetectorConfig::chess()` (deprecated alias remains) |
| `DetectorConfig::multiscale()` | `DetectorConfig::chess_multiscale()` (deprecated alias remains) |
| Field-mutation refiner swap (4 lines) | `.with_chess(&#124;c&#124; c.refiner = ChessRefiner::forstner())` |
| `ChessRefiner::Forstner(ForstnerConfig::default())` | `ChessRefiner::forstner()` |
| `MultiscaleConfig::Pyramid { levels: 3, min_size: 128, refinement_radius: 3 }` | `MultiscaleConfig::pyramid_default()` |

Python:

```text
cfg.strategy.chess.refiner = ChessRefiner.forstner(...)   → cfg.with_chess(refiner=ChessRefiner.forstner())
```

JS:

```text
cfg.strategy.chess.refiner = ChessRefiner.withForstner({})  → cfg.withChess({ refiner: ChessRefiner.withForstner({}) })
```

## Breaking changes

- `DetectorConfig` is the only public configuration type. The
  deprecated `ChessConfig = DetectorConfig` alias is removed.
- The strategy payload types are `ChessConfig` and `RadonConfig`.
- Multiscale knobs live on `MultiscaleConfig` (an enum, not an
  optional struct).
- `UpscaleConfig` is an enum (`Disabled | Fixed(u32)`), not a
  two-field struct.
- The top-level refiner / descriptor-mode fields are removed.
  Refiners and descriptor-ring choice live inside the strategy
  payload.
- Python: the free-function `find_chess_corners` is removed in
  favour of `Detector(cfg).detect(img)`. The `DetectorMode` /
  `ThresholdMode` / `RefinementMethod` Python enums are removed; use
  the tagged classes `Threshold`, `ChessRefiner`, `RadonRefiner`.
- WASM: the imperative shortcuts on `ChessDetector`
  (`set_detector_mode`, `set_broad_mode`, `set_nms_radius`,
  `set_min_cluster_size`, `set_threshold`, `set_pyramid_levels`,
  `set_min_pyramid_size`, `set_refinement_radius`) are removed. Edit
  the typed config and call `applyConfig` instead.

## Migration

Rust:

```text
ChessConfig                          → DetectorConfig
ChessStrategy                        → ChessConfig
RadonStrategy                        → RadonConfig
MultiscaleParams { ... }             → MultiscaleConfig::Pyramid { ... }
Option<MultiscaleParams>::None       → MultiscaleConfig::SingleScale
UpscaleConfig { mode, factor }       → UpscaleConfig::Fixed(factor) / Disabled
cfg.refiner.kind = X                 → cfg.strategy.<variant>.refiner = X(_)
cfg.descriptor_mode                  → cfg.strategy.chess.descriptor_ring
```

Python:

```text
DetectorConfig.multiscale()          → DetectorConfig.chess_multiscale()
cfg.detector_mode                    → cfg.strategy.chess.ring
cfg.refiner.kind                     → cfg.strategy.chess.refiner.kind (read)
                                       cfg.strategy.chess.refiner = ChessRefiner.forstner(...) (write)
cfg.descriptor_mode                  → cfg.strategy.chess.descriptor_ring
threshold_mode / threshold_value     → cfg.threshold = Threshold.absolute(v) / .relative(f)
```

JS:

```text
detector.set_detector_mode(mode)     → detector.applyConfig(cfg) after editing cfg.strategy
detector.set_broad_mode(true)        → cfg.strategy.chess.ring = ChessRing.Broad
detector.set_threshold(value)        → cfg.threshold = Threshold.absolute(value)
detector.set_pyramid_levels(n)       → cfg.multiscale = MultiscaleConfig.pyramid(n, min, r)
ChessRefiner with forstner config    → ChessRefiner.withForstner(cfg)
```
