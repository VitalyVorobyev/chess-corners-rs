# 0.11.0 — 2026-05-17

A breaking release that tightens the public API surface of both the
`chess-corners` facade and the `chess-corners-core` crate. Normal
detection code is unaffected; only callers of diagnostic outputs,
low-level pipeline stages, or internal stage-module paths need to
migrate.

## Changed

- `chess-corners-ml` is now marked `publish = false`; it is an internal
  implementation crate of this workspace (backing the `ml-refiner` feature
  of `chess-corners`) and is not a crates.io publication.

- **Breaking:** The `chess-corners` crate root now surfaces only the
  stable facade — the detector, its configuration and result types, and
  errors. Diagnostic outputs (`ResponseMap`, `chess_response_u8`,
  `chess_response_u8_patch`, `radon_heatmap_u8`, `radon_heatmap_image`)
  are now reachable via `chess_corners::diagnostics`. Low-level pipeline
  stages, parameter structs, and scratch buffers (`chess_response_u8`,
  `detect_corners_from_response`,
  `detect_corners_from_response_with_refiner`, `describe_corners`,
  `Corner`, `Roi`, `ChessParams`, `RadonDetectorParams`,
  `CoarseToFineParams`, `ImageView`, `RadonBuffers`, `UpscaleBuffers`,
  `ImageBuffer`, `PyramidBuffers`, `PyramidParams`, `Refiner`,
  `RefinerKind`, `CornerRefiner`, `RefineResult`, `RefineStatus`,
  `upscale_bilinear_u8`, `rescale_descriptors_to_input`) are now
  reachable via `chess_corners::low_level`. These items are no longer
  re-exported from the crate root.
- **Breaking:** The `chess-corners-core` crate now exposes a deliberate
  low-level contract through its crate root instead of leaving the
  detector's stage modules public. The `detect`, `orientation`,
  `refine`, and `imageview` modules are now private; the supported
  low-level surface (response computation, corner detection, refiners,
  the orientation fit, `ImageView`, parameter and result types) remains
  reachable directly from the `chess_corners_core` crate root. Code
  that imported through stage module paths (for example
  `chess_corners_core::detect::chess::response::chess_response_u8`)
  should import the same item from the crate root instead.
- **Breaking:** Implementation primitives that have no semver guarantee
  — ring offset tables (`RING5`, `RING10`, `ring_offsets`,
  `RingOffsets`), the Radon angular basis (`ANGLES`, `DIR_COS`,
  `DIR_SIN`), `box_blur_inplace`, `fit_peak_frac`, `SatElem`,
  `MAX_IMAGE_UPSAMPLE`, the scalar reference response path
  (`chess_response_u8_scalar`), `chess_response_u8_patch`, and the
  sub-stage detection entry points — now live in a new
  `chess_corners_core::unstable` module. Items in `unstable` are
  exposed for benchmarks, experiments, and advanced use and may change
  or be removed in any release.
- The `AxisFitResult` type is no longer re-exported from the
  `chess-corners` crate root; it is available via
  `chess_corners::low_level` alongside `fit_axes_at_point`.
- **Breaking (WASM/JS):** The diagnostic methods on the JavaScript
  `ChessDetector` are now prefixed with `diagnostics_` so they read
  clearly as opt-in debugging and visualization data rather than
  primary detection output. Rename calls: `response` →
  `diagnostics_response`, `response_rgba` → `diagnostics_response_rgba`,
  `response_width` → `diagnostics_response_width`, `response_height` →
  `diagnostics_response_height`, `radon_heatmap` →
  `diagnostics_radon_heatmap`, `radon_heatmap_rgba` →
  `diagnostics_radon_heatmap_rgba`, `radon_heatmap_width` →
  `diagnostics_radon_heatmap_width`, `radon_heatmap_height` →
  `diagnostics_radon_heatmap_height`, `radon_heatmap_scale` →
  `diagnostics_radon_heatmap_scale`. The `detect` and `detect_rgba`
  methods are unchanged.

## Removed

- **Breaking:** Removed the `Detector::radon_heatmap_u8` and
  `Detector::radon_heatmap` methods. Use
  `Detector::diagnostics().radon_heatmap_u8(...)` /
  `Detector::diagnostics().radon_heatmap(...)` instead.
- **Breaking (Python):** Removed the deprecated
  `DetectorConfig.single_scale()` and `DetectorConfig.multiscale_preset()`
  aliases. Use `DetectorConfig.chess()` and
  `DetectorConfig.chess_multiscale()` instead.

## Migration

### Normal detection — unchanged

```rust
use chess_corners::{Detector, DetectorConfig};

let cfg = DetectorConfig::chess();
let mut detector = Detector::new(cfg)?;
let corners = detector.detect(&gray_image)?;
```

`Detector`, `DetectorConfig`, and `CornerDescriptor` are all still at
the crate root and their signatures are unchanged.

### Rust: diagnostics (response maps and heatmaps)

Before:

```rust
use chess_corners::{Detector, chess_response_u8, ResponseMap};

let map: ResponseMap = chess_response_u8(&pixels, w, h, &params);
let heatmap = detector.radon_heatmap_u8(&pixels, w, h)?;
```

After:

```rust
use chess_corners::{Detector, diagnostics};

let map = diagnostics::chess_response_u8(&pixels, w, h, &params);
let heatmap = detector.diagnostics().radon_heatmap_u8(&pixels, w, h)?;
```

### Rust: low-level pipeline composition

Before:

```rust
use chess_corners::{
    Corner, ChessParams, detect_corners_from_response,
    PyramidBuffers, Refiner, RefinerKind,
};
```

After:

```rust
use chess_corners::low_level::{
    Corner, ChessParams, detect_corners_from_response,
    PyramidBuffers, Refiner, RefinerKind,
};
```

### Rust: chess-corners-core stage-module paths

Before:

```rust
use chess_corners_core::detect::chess::response::chess_response_u8;
use chess_corners_core::orientation::describe_corners;
```

After:

```rust
use chess_corners_core::{chess_response_u8, describe_corners};
```

Implementation primitives with no semver guarantee (ring tables,
Radon angular bases, `box_blur_inplace`, `fit_peak_frac`,
`MAX_IMAGE_UPSAMPLE`, `chess_response_u8_patch`, etc.) have moved to
`chess_corners_core::unstable`:

```rust
use chess_corners_core::unstable::{MAX_IMAGE_UPSAMPLE, fit_peak_frac};
```

### WASM/JS: diagnostic method names

Rename all diagnostic calls with the new `diagnostics_` prefix:

| Before | After |
|--------|-------|
| `detector.response(pixels, w, h)` | `detector.diagnostics_response(pixels, w, h)` |
| `detector.response_rgba(pixels, w, h)` | `detector.diagnostics_response_rgba(pixels, w, h)` |
| `detector.response_width()` | `detector.diagnostics_response_width()` |
| `detector.response_height()` | `detector.diagnostics_response_height()` |
| `detector.radon_heatmap(pixels, w, h)` | `detector.diagnostics_radon_heatmap(pixels, w, h)` |
| `detector.radon_heatmap_rgba(pixels, w, h)` | `detector.diagnostics_radon_heatmap_rgba(pixels, w, h)` |
| `detector.radon_heatmap_width()` | `detector.diagnostics_radon_heatmap_width()` |
| `detector.radon_heatmap_height()` | `detector.diagnostics_radon_heatmap_height()` |
| `detector.radon_heatmap_scale()` | `detector.diagnostics_radon_heatmap_scale()` |

The `detect` and `detect_rgba` methods are unchanged.

### Python: removed constructor aliases

Before:

```python
cfg = DetectorConfig.single_scale()
cfg = DetectorConfig.multiscale_preset()
```

After:

```python
cfg = DetectorConfig.chess()
cfg = DetectorConfig.chess_multiscale()
```

Python stubs now include `Detector.config()` and
`Detector.apply_config()` for reading and updating the active
configuration at runtime.
